Task Manager: React Component Architecture Demo

A single-page application (SPA) built to demonstrate modern front-end development using React, hooks (useState, useEffect) and responsive utility-first styling with Tailwind CSS.

React JS Tailwind CSS State Management Dark Mode

Technical Features and Architecture

The application is structured around reusable components (Buttons, Cards, Navigation) to ensure scalability and maintainability. A key feature is the Dark/Light Mode toggle, implemented using React's Context API for global state management.

I utilized functional components and Hooks like `useState` to manage local task lists and `useEffect` for handling side effects, such as API data fetching (simulated or real). The styling is fully responsive, ensuring optimal display on mobile, tablet and desktop devices using Tailwind's utility classes.

The core logic lies in the Task Manager component, which handles the creation, listing, filtering and deletion of tasks, demonstrating proficiency in common application logic and form handling in React.

Development Breakdown

1

Component Structure & Setup

Initialized the React app and configured Tailwind CSS. Created initial components: NavBar, Footer, and the main Task list container.

2

State Management Implementation

Developed the Context Provider for the Dark Mode feature. Used the `useContext` hook to enable seamless theme switching across all components.

3

Task Logic (CRUD & Filtering)

Implemented functions using `useState` to handle adding new tasks, deleting existing ones, and filtering the list based on search input or status.

4

API Integration & Final Styling

Configured `useEffect` to fetch initial data from a mock API. Finalized responsive design using Tailwind's breakpoints.

Key Technologies Utilized

This project emphasized modern front-end best practices, focusing on performance and modularity through component-based architecture and utility styling.
